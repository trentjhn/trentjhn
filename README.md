## Live builds

### [PQC Deal Engine](https://pqc-deal-engine.vercel.app/)
Generates a board-ready post-quantum-cryptography deal readout for any company: harvest-now-decrypt-later risk quantified with the Mosca inequality, the exact regulatory deadlines their vertical faces, a NIST-suite migration sketch, and a CISO one-pager. Every fact is grounded in a versioned data file so it can't be hallucinated. Built and deployed in a day.

### [GitRecap](https://gitrecap-gamma.vercel.app)
Reconstructs what you did each day from your GitHub commits and translates it to plain English, so reviewing your work is reflection, not log-reading. Next.js + Claude Haiku, with a frozen-past caching design that runs at near-zero cost. (Access-gated demo.)

### [Viridian](https://github.com/trentjhn/viridian)
A Go terminal UI that watches Claude Code sessions in real time: every tool call, token spend, session memory, and file diff, via the host harness's hook system. A tool about the AI harness, not one built by it.

### [quantum-arxiv-digest](https://github.com/trentjhn/quantum-arxiv-digest)
A clone-and-run CLI that pulls the latest quantum, post-quantum-cryptography, and cryptography papers off arxiv into clean, browsable folders. Runs with no API key.

## How I build

### [Builds Log](https://github.com/trentjhn/builds)
The full record of the AI systems I've shipped, the architecture decisions behind each, and the patterns running through all of them: state as files instead of chat memory, grounding gates that make a wrong fact structurally impossible, and math offloaded to deterministic code.

### [AI-Knowledgebase](https://github.com/trentjhn/AI-Knowledgebase)
Most AI engineering tutorials are either too shallow for practitioners or benchmark-focused and decay in months. This is a practitioner-depth synthesis built to age well: 14+ docs across prompting, context engineering, agentic systems, evaluation, fine-tuning, and security, structured as a learning path from first principles to production.

It doubles as an operational system: an agent catalog, reusable skills, playbooks, and `/cook`, a 9-phase project scaffold workflow that turns the knowledge base into a decision engine for any new build. Weekly ArXiv digests auto-ingest current research.
